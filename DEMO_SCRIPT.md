# 易挂念 5 分钟演示脚本

> 适用场景：2026/07/20 溪东社区现场评审

---

## 准备工作

```bash
cd backend
PYTHONPATH=. python seed_elders.py    # 2101 位老人 + 15 名工作者
PYTHONPATH=. python seed_demo.py      # 演示场景数据
```

管理后台登录：`13800138000 / admin123`

---

## 场景一：看板概览（60s）

**操作路径**：管理后台 → 看板

1. 打开管理后台看板页
2. 指出顶部 6 个统计卡片：总人数 2101、A/B/C 级分布、今日活跃、活跃率
3. 点击「全屏展示」进入大屏模式
   - 顶部横幅：社区名称 + 实时数据 + 时钟
   - 楼栋热力图：颜色深浅表示活跃率
   - 指出祥盛家园 3 号楼有告警标记
4. ESC 退出全屏

**话术**：「这是社区工作人员每天打开的第一个页面。2101 位老人的状态一目了然。颜色越深代表越活跃，红色楼栋需要关注。」

---

## 场景二：张奶奶的故事（90s）

**操作路径**：事件中心 → 找到张奶奶告警 → 老人详情

1. 事件中心页面指出张奶奶的告警：「超过18小时无活动信号」
2. 点击进入老人详情页
   - 基本信息：B 级，祥盛家园 3 号楼，独居
   - 时间线：29 天都有查看记录、食堂出勤
   - 今天：没有查看、午餐缺席 → 系统自动产生告警
3. 展示 7 天趋势图：前几天正常，今天断了
4. AI 分析（如有接入）：综合判断异常

**话术**：「张奶奶独居，平时每天都会看女儿发的牵挂内容，也准时去食堂吃饭。今天两个信号都断了，系统自动预警通知网格员。不是监控，是关心——已读即在线。」

---

## 场景三：家属端（60s）

**操作路径**：小程序 → 女儿张小红视角

1. 打开小程序首页（家属模式）
   - 展示绑定的老人卡片
2. 进入「发送牵挂」页面
   - 选择照片 → 写一句话 → AI 帮我写（可选）
   - 发送后弹出订阅消息授权
3. 切换到老人视角
   - 打开小程序卡片看到牵挂内容（大字体、高对比度）
   - 点击「我很好」报平安 → 按钮变绿
4. 告警页面：查看提醒 → 点击「已确认安好」

**话术**：「子女只需要分享日常——拍张照片、写句话。老人打开看了就代表在线。如果迟迟没看，系统会温柔地提醒子女。老人端零学习成本，点开就能看。」

---

## 场景四：食堂闭环（60s）

**操作路径**：管理后台 → 食堂管理

1. 展示食堂管理页面
   - 已有的出勤记录列表
   - 点击一条记录查看解析结果：到场/缺席名单
   - A/B 级缺席标红
2. 指出张奶奶缺席 → 事件中心已自动产生事件
3. 处理事件：填写处理说明 → 关闭

**话术**：「食堂是社区观测老人状态的核心窗口。工作人员提交出勤数据后，系统自动识别缺席的重点老人并产生事件，形成闭环。」

---

## 场景五：邻里帮积分（30s）

**操作路径**：管理后台 → 邻里帮

1. 展示三个 Tab
   - 任务管理：已完成 2 个、待接 3 个
   - 志愿者名单：3 名 C 级老人注册
   - 积分排行：前三名有奖杯标记
2. 小程序志愿者页面（可选）
   - 注册志愿者 → 查看可接任务 → 接单

**话术**：「健康的 C 级老人可以成为志愿者，帮助需要照顾的老人。探访一次 15 积分，积分可以兑换社区服务。互助养老，社区自治。」

---

## 场景六：数据导出（30s）

**操作路径**：管理后台 → 老人档案 / 事件中心

1. 老人档案页点击「导出 Excel」→ 下载 .xlsx
2. 打开文件：姓名、等级、地址、风险分、联系人、网格员
3. 事件页也可以导出

**话术**：「所有数据支持一键导出 Excel，方便汇报和存档。」

## 总结（30s）

**话术**：
「易挂念不是监控系统，是一个用牵挂连接家人的平台。

三个核心理念：

- **社区智能协同**——AI 辅助判断、自动预警、闭环处理

覆盖溪东社区 2101 位老人，15 名工作人员日常使用。」
