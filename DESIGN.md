# EZLove 易挂念 — 设计系统

## 品牌哲学

**「让关心自然流动，让牵挂被看见」**

EZLove 不是一个监控系统，而是一个关爱平台。所有设计决策必须服务于这个核心：温暖、尊严、自然。

- 老人感知到的是家人的牵挂，而非被监视
- 社区工作人员看到的是关爱的流动，而非冰冷的数据
- 子女体验到的是安心的连接，而非焦虑的监控

## 风格基调：质朴简约 · 高级感

**关键词**：质朴 · 克制 · 温度 · 精致

### 视觉原则

1. **留白即高级** — 大量负空间，让内容自己说话。不堆砌元素，不滥用装饰
2. **色彩克制** — 主色不超过3种，辅以中性色系。用色温传递情感，而非色彩饱和度
3. **细节见品质** — 1px 的边框、微妙的阴影、精心选择的圆角、考究的字体搭配
4. **数据即艺术** — 数据可视化不是图表堆砌，而是用优雅的方式让信息自然呈现
5. **动效有温度** — 动画服务于体验，不为炫技。柔和的缓动、有节奏的出场

### 禁止事项

- ❌ 不使用渐变炫彩、霓虹发光、玻璃拟态
- ❌ 不使用饱和度 > 70% 的大面积色彩
- ❌ 不使用卡通/可爱风格的图标或插画
- ❌ 不使用系统默认字体（Arial, Helvetica, system-ui）
- ❌ 不使用紫色渐变、蓝色渐变等"AI味"配色
- ❌ 不使用粗重的投影和过度的层级效果

## 动效系统

### 缓动曲线

| 名称 | 值 | 用途 |
|------|-----|------|
| ease-out | `cubic-bezier(0.16, 1, 0.3, 1)` | 元素进入 |
| ease-in-out | `cubic-bezier(0.4, 0, 0.2, 1)` | 状态切换 |

### 时长

| 名称 | 值 | 用途 |
|------|-----|------|
| fast | 150ms | 按钮悬停、输入焦点 |
| normal | 250ms | 页面过渡、展开收起 |
| slow | 400ms | 页面加载、大区块动画 |

### 入场动画

- **fadeInUp**: 从下方12px渐入，用于卡片和区块
- **stagger**: 子元素依次入场，间隔60ms，用于列表和卡片组
- **page-fade**: 页面切换时渐入渐出

## 数据可视化准则

### 核心原则：数据即艺术

数据可视化不是技术图表，而是信息设计。每一个数据展示都应该像一件精心编排的作品。

### 统计卡片

- 图标 + 数字 + 标签的三段式结构
- 图标使用品牌色系的柔和背景色圆角方块
- 数字使用 Display 字体（霞鹜文楷），传递温度感
- 卡片悬停时微微上浮（translateY -2px）+ 阴影加深
- 入场时从下方渐入，子卡片依次错开 60ms

### 热力图

- 按楼栋/区域分组，每组有标签和人数统计
- 每个人用一个 52×52px 的方块表示，显示姓氏首字
- 状态用色彩编码：活跃=鼠尾草绿渐变，A级未活跃=赤陶红渐变（带脉冲动画），未活跃=暖石灰渐变
- 方块间距 8px，悬停时放大 1.1 倍
- 分组卡片使用奶油白背景 + 暖沙边框

### 表格

- 表头使用宣纸背景、小号大写字母、烟灰色
- 行使用白色背景，悬停时奶油白
- 姓名列带头像（首字母圆形，分级色彩背景）
- 分级标签使用药丸形状 + 对应背景色
- 状态使用圆点 + 文字的轻量组合

### 图表（未来）

- 折线图：使用品牌色，线条 2px，数据点 6px 圆点
- 柱状图：圆角顶部，品牌色系渐变
- 饼图/环形图：最多5个分类，使用品牌色板
- 所有图表必须有入场动画（从0到实际值的过渡）

## 页面布局准则

### Web 端（管理后台）

```
┌──────────────────────────────────────┐
│ 侧边栏(240px)  │  内容区             │
│ ┌────────────┐ │ ┌─────────────────┐ │
│ │ Logo       │ │ │ 面包屑    用户  │ │ ← 顶栏 56px
│ ├────────────┤ │ ├─────────────────┤ │
│ │ 📋 看板    │ │ │                 │ │
│ │ 👥 档案    │ │ │   页面内容      │ │
│ │ 🍽 食堂    │ │ │                 │ │
│ │ 🔔 事件    │ │ │                 │ │
│ ├────────────┤ │ │                 │ │
│ │ [折叠]     │ │ │                 │ │
│ └────────────┘ │ └─────────────────┘ │
└──────────────────────────────────────┘
```

- 侧边栏：深色（炉黑 #2C2825），导航项有选中态（赤陶底 + 左侧竖条）
- 顶栏：白色背景，面包屑 + 用户头像
- 内容区：奶油白背景，24px 内边距
- 页面切换：fade 过渡动画

### 小程序端

- 底部 Tab 导航：首页、记录、提醒、我的
- 页面背景：奶油白 `#FFF8F0`
- 卡片：白色背景，24rpx 圆角，微妙投影
- 按钮：主色赤陶橙 `#FF8C42`，48rpx 圆角
- 长辈端：最小字号 36rpx，按钮最小高度 96rpx

---

## 图标风格

- 使用线性图标（stroke），不用填充图标（fill）
- 线宽 1.5px，端点圆润
- 图标尺寸：导航 20px，正文 16px，装饰 24px
- 图标颜色跟随文本颜色，不使用独立色彩

## 文案语调

### 原则

- **温暖不矫情** — 用朴实的语言传递关心，不用华丽辞藻
- **专业不冰冷** — 技术术语转化为用户能理解的表达
- **简洁不敷衍** — 每个字都有存在的理由

### 示例

| 场景 | ✅ 推荐 | ❌ 避免 |
|------|---------|---------|
| 老人未读 | "今天还没有查看消息" | "用户未活跃" |
| 预警 | "张大爷超过12小时未查看" | "触发 unread alert" |
| 分级 | "A级 · 重点关爱" | "care_level: A" |
| 操作成功 | "已添加" | "创建成功！" |
| 空状态 | "还没有老人数据，点击添加第一位" | "暂无数据" |
